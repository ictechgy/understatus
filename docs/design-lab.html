Design notes
Glyph strategy. Emoji faces replaced with five minimal Unicode characters: · (idle) ─ (light) ▸ (moderate) ▲ (elevated) ◆ (critical). Each is one character wide, readable in any terminal font, has no color of its own, and escalates visually along a purely geometric axis — not an emotional one.
  • States 0–3 use only cool blue-grays, shifting temperature/brightness by ~15 L* steps. No hue shift, no saturation increase. A developer reading code will not register these as "alerts".
  • State 4 is the single hue exception: amber #c48f5a — warm but desaturated, closer to an aged document than a warning light. The slow opacity pulse (50s default / 5s demo) is perceptible on the periphery without demanding attention.
  • OMC HUD tail is rendered at 28% opacity with a hairline left border, making it clearly separate yet readable when glanced at. understatus's own segment never competes.
  • Separators are mid-dot · at near-invisible gray rather than pipes or slashes, keeping the baseline flat and undemanding.
  • Label/value split: short prefixes (mem disk ctx ⎇) at the dimmest text tier; values one tier brighter. The hierarchy reads in a single glance.
